1.2 Family Size¶

Taking a look at the data dictionary, we see there are two columns, SibSp and Parch. SibSp represents the number of siblings/spouses aboard and Parch represents the number of parents/children aboard Let's create a new variable called Fsize that represents the size of the family, using the formula 1 + SibSp + Parch,

To explain the above formula, take one observation, say the first one.

The name of the person is Braund, Mr. Owen Harris, and he has 1 under the SibSp and 0 under the Parch column, so his family size will be himself + SibSp + Parch, which translates to 1 + SibSp + Parch

def missing_values_table(titanic): 
        mis_val = titanic.isnull().sum()
        mis_val_percent = 100 * titanic.isnull().sum()/len(titanic)
        mis_val_table = pd.concat([mis_val, mis_val_percent], axis=1)
        mis_val_table_ren_columns = mis_val_table.rename(
        columns = {0 : 'Missing Values', 1 : '% of Total Values'})
        return mis_val_table_ren_columns

def graph_feature_importances(model, feature_names, autoscale=True,
                              headroom=0.05, width=10, summarized_columns=None):
    
    if autoscale:
        x_scale = model.feature_importances_.max()+ headroom
    else:
        x_scale=1
        
    feature_dict=dict(zip(feature_names, model.feature_importances_))
    
    if summarized_columns:
        
        for col_name in summarized_columns:
            
            sum_value = sum(x for i, x in feature_dict.iteritems() if col_name in i)
            
            keys_to_remove = [i for i in feature_dict.keys() if col_name in i]
            for i in keys_to_remove:
                feature_dict.pop(i)
                
            feature_dict[col_name] = sum_value
            
    results = pd.Series(feature_dict.values(), index=feature_dict.keys())
    
    results = results.sort_values(ascending=True)
    
    ax = results.plot(kind="barh", figsize=(width,len(results)/4), xlim=(0,x_scale))
    ax.set_xlabel('Importance') 
    ax.set_ylabel('Features')
    plt.title('Variable Importance')

5. Limitations¶

  1. 77% of missing values in the 'Cabin' column was obviously something you don't want. Knowing which deck the person was in could be an explanatory variable. For example, the first chatacter from the 'Cabin' column shows the deck. If the person was at a lower deck, maybe he could avail the lifeboat service easier than someone who was in the 'F' deck (topmost) deck, and it was harder for him to avail the lifeboat service.

  2. Knowledge of Missing Value Imputation techniques: Knowing which method of imputation would give you the best results. Beginners tend to just fill missing values with mean or medians, but a lot of other advaced packages exist for missing value imputations.

  3. Choice of Programming Language: Using R instead of Python would've been more easier for Data Analysis due to the abundance of statistical packages. For examaple, the MICE package for Missing Value Imputation exists in R but not in Python, sure if you're skilled enough, you could read the research paper and write your own function, but beginners like me don't know how to go about it.

  4. Not having the entire dataset to perform a more thorough analysis. Only 40% of data was used to perform analysis. Then again, this was originally posted on kaggle.com as a Machine Learning competition so maybe the remaining 60% was for testing the built ML model.

6. Summaries and Conclusions¶

  1. No. of deceased: 549, No. of survivors: 342, however, only data of 891 out of 2224 was used for analysis.

  2. There could be more explanatory variables that explain the survival status of the individual, maybe if there was a variable which depicted a person's ability to swim. Surely, after the crash, some may have drowed straight away and some being good swimmers managed to scramble into a lifeboat. Also, we are only able to infer family sizes but more in depth analysis can be performed if another variable depicted if a certain member was a part of a certain family. This could have been achieved by indexing each member with a family number or maybe that information is already available in the data and I didn't see it.

  3. The original data had 687 missing 'Cabin' details, 177 missing 'Age' details and 2 'Embarked Port' details. The Cabin column was simply dropped due to 77% of them missing, the missing values in 'Age' and the 'Embarked' were imputed.

  4. The Machine Learning model used in this assignment (Random Forest) was just one of the many Supervised ML Models there is to make predictions. There are many evaluation metrics. One of them used here was the 'ROC_AUC_SCORE'. There are other metrics such as Precision, Recall and others. By no means does one evaulation metric like the one I used say that this model is reliable.

The 'Ideal' metric score depends on the context of the problem that you are trying to solve. If the emphasis is on making correct predictions (that is, when you predict that something is true, then it is), you want to maximize precision (so, in this case, you don't mind missing some cases where it is true (False Negatives), just that when you say it is true then it is). If the emphasis is on making sure that when something is true that you predict it is, then you want to maximize recall (so, in this case, you don't mind predicting cases that aren't true as true (False Positives), just that you catch all of the cases that are true).
